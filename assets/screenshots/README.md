## These are designed placeholders

The committed `.svg` files are **on-brand mockups** generated by
[`scripts/gen-screenshots.mjs`](../../scripts/gen-screenshots.mjs) so the README and website
render correctly out of the box. Regenerate them any time with:

```bash
node scripts/gen-screenshots.mjs
```

## Replacing them with real captures

For the most polished look, replace these with real screenshots of the running app:

1. Open the app (or the live demo at <https://grids.swyftstack.com/demo>) in **dark mode**.
2. Capture each view at a high resolution — **2560 × 1600** (a 2× "Retina" frame) works well.
3. Crop to the app window (no OS chrome) and export as **PNG**.
4. Save them here using the same base names with a `.png` extension
   (`hero.png`, `sql-editor.png`, …).
5. Update the image links in the root `README.md` from `.svg` to `.png`.

> Keep individual files under ~1 MB so the README stays fast to load. Optimize PNGs with
> `oxipng` / `pngquant` if needed.
